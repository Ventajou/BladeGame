Break time

When our games run in a desktop browser, we can expect our players to switch to a different window every once in a while. When they do, they will appreciate to see the game automatically pause itself. Not only will this behavior increase the chances that the player will return to the game; it will also keep from eating CPU cycles and eventually battery life.

By properly separating the code, we can also ensure that we avoid artifacts on the screen. The trick being to refresh the display while not updating the state of the game.
